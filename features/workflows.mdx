## Overview

Workflows in Eru allow you to chain multiple requests together, extract data from responses using JSONPath, and use that data in subsequent requests. This is essential for testing complex API flows like authentication → data creation → data retrieval → cleanup.

<Info>
Workflows are stored in browser localStorage and can be saved, loaded, and executed multiple times with full step-by-step visibility.
</Info>

## Workflow Structure

A workflow consists of multiple steps executed sequentially:

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:31-38
interface Workflow {
  id: string;
  name: string;
  description?: string;
  steps: WorkflowStep[];
  createdAt: number;
  updatedAt: number;
}
```

### Workflow Step

Each step represents a single request with variable extraction:

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:15-29
interface WorkflowStep {
  id: string;
  name: string;
  requestId: string;     // Reference to existing request
  order: number;
  extractVariables: WorkflowVariable[];
  // Overrides - these override the original request values
  bodyOverride?: string;          // Override request body
  headersOverride?: Record<string, string>;  // Override/add headers
  urlOverride?: string;           // Override URL (optional)
  // Execution state
  status?: "pending" | "running" | "success" | "failed" | "skipped";
  response?: Response;
  error?: string;
}
```

## Variable Extraction

Extract data from responses using JSONPath expressions:

### JSONPath Syntax

<CodeGroup>
```javascript Root Object
// Extract from root level
"$.token"           // Extracts: response.token
"$.user.id"         // Extracts: response.user.id
"$.data.attributes" // Extracts: response.data.attributes
```

```javascript Array Access
// Extract from arrays
"$.items[0].id"        // First item's ID
"$.users[2].name"      // Third user's name
"$.data.products[0]"   // First product
```

```javascript Nested Objects
// Complex nested paths
"$.response.data.user.profile.avatar"
"$.results[0].metadata.createdAt"
```
</CodeGroup>

### JSONPath Implementation

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:48-71
function extractJsonPath(obj: unknown, path: string): string | undefined {
  if (!path.startsWith("$")) return undefined;
  
  const parts = path.slice(2).split(/\.|\[|\]/).filter(Boolean);
  let current: unknown = obj;
  
  for (const part of parts) {
    if (current === null || current === undefined) return undefined;
    if (typeof current !== "object") return undefined;
    
    const index = parseInt(part, 10);
    if (!isNaN(index) && Array.isArray(current)) {
      current = current[index];
    } else {
      current = (current as Record<string, unknown>)[part];
    }
  }
  
  if (current === null) return "null";
  if (typeof current === "object") return JSON.stringify(current);
  return String(current);
}
```

### Variable Interpolation

**Standard Interpolation (URLs, Headers):**

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:73-82
function interpolateVariables(
  text: string,
  variables: Map<string, string>
): string {
  return text.replace(/\{\{([^}]+)\}\}/g, (match, varName) => {
    const value = variables.get(varName.trim());
    return value !== undefined ? value : match;
  });
}
```

**JSON-Aware Interpolation (Request Body):**

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:85-102
function interpolateVariablesJson(
  text: string,
  variables: Map<string, string>
): string {
  return text.replace(/\{\{([^}]+)\}\}/g, (match, varName) => {
    const value = variables.get(varName.trim());
    if (value === undefined) return match;

    // If value is already valid JSON (number, boolean, null, object, array), use as-is
    try {
      JSON.parse(value);
      return value;
    } catch {
      // Not valid JSON, treat as raw string and JSON-stringify it
      return JSON.stringify(value);
    }
  });
}
```

<Tabs>

    ```json
      "category": "electronics"
    }
    ```
    
    **Use Case:** Testing different scenarios with the same request template
  </Tab>
  
  <Tab title="Headers Override">
    ### Add/Override Headers
    
    Add workflow-specific headers or override existing ones:
    
    ```json
    {
      "Authorization": "Bearer {{Login.token}}",
      "X-Test-Run": "workflow-execution",
      "X-Step-ID": "create-product"
    }
    ```
    
    **Merge Behavior:**
    - Original request headers are preserved
    - Override headers are added or replace matching keys
  </Tab>
  
  <Tab title="URL Override">
    ### Override URL
    
    Replace the entire URL for a specific step:
    
    ```
    {{Login.baseUrl}}/v2/products/{{CreateProduct.productId}}
    ```
    
    **Use Case:** Testing different API versions or endpoints
  </Tab>
</Tabs>

### Execution Implementation

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:1462-1557
const executeWorkflow = async () => {
  if (isRunning()) return;
  
  setIsRunning(true);
  setExecutionLog([]);
  setCollectedVariables(new Map());
  
  // Reset all step statuses
  const w = workflow();
  setWorkflow({
    ...w,
    steps: w.steps.map(s => ({ ...s, status: "pending", response: undefined, error: undefined })),
  });

  log(`Starting workflow: ${w.name}`, "info");

  const variables = new Map<string, string>();

  for (let i = 0; i < w.steps.length; i++) {
    const step = workflow().steps[i];
    const request = requestMap().get(step.requestId);
    
    if (!request) {
      updateStep(step.id, { status: "failed", error: "Request not found" });
      log(`Step ${i + 1} failed: Request not found`, "error");
      continue;
    }

    updateStep(step.id, { status: "running" });
    log(`Running step ${i + 1}: ${step.name}`, "info");

    try {
      // Build request with step overrides first
      const baseRequest: Request = {
        ...request,
        url: step.urlOverride || request.url,
        body: step.bodyOverride !== undefined ? step.bodyOverride : request.body,
        headers: step.headersOverride 
          ? { ...request.headers, ...step.headersOverride }
          : request.headers,
      };

      // Then interpolate variables (use JSON-aware interpolation for body)
      const interpolatedRequest: Request = {
        ...baseRequest,
        url: interpolateVariables(baseRequest.url, variables),
        body: baseRequest.body ? interpolateVariablesJson(baseRequest.body, variables) : undefined,
        headers: Object.fromEntries(
          Object.entries(baseRequest.headers).map(([k, v]) => [k, interpolateVariables(v, variables)])
        ),
      };

      // Execute the request
      const response = await props.onExecuteRequest(interpolatedRequest);
      
      updateStep(step.id, { status: "success", response });
      log(`Step ${i + 1} completed: ${response.status} (${response.duration}ms)`, "success");

      // Extract variables from response
      if (step.extractVariables.length > 0) {
        try {
          const responseBody = JSON.parse(response.body);
          
          for (const variable of step.extractVariables) {
            const value = extractJsonPath(responseBody, variable.jsonPath);
            if (value !== undefined) {
              const fullName = `${step.name}.${variable.name}`;
              variables.set(fullName, value);
              
              // Also update the variable value in the step for display
              updateVariable(step.id, variable.id, { value });
              log(`  Extracted {{${fullName}}} = ${value.substring(0, 50)}${value.length > 50 ? "..." : ""}`, "info");
            } else {
              log(`  Warning: Could not extract ${variable.name} from ${variable.jsonPath}`, "error");
            }
          }
        } catch (e) {
          log(`  Warning: Response body is not valid JSON`, "error");
        }
      }

      setCollectedVariables(new Map(variables));

    } catch (error) {
      const errorMsg = error instanceof Error ? error.message : "Unknown error";
      updateStep(step.id, { status: "failed", error: errorMsg });
      log(`Step ${i + 1} failed: ${errorMsg}`, "error");
    }

    // Small delay between steps for visibility
    await new Promise(resolve => setTimeout(resolve, 300));
  }

  log(`Workflow completed`, "success");
  setIsRunning(false);
};
```

## Workflow Storage

Workflows are stored in browser localStorage:

```typescript
// From: apps/desktop/ui/src/components/WorkflowEditor.tsx:1175-1199
const WORKFLOWS_STORAGE_KEY = "api_client_workflows";

const loadSavedWorkflows = () => {
  try {
    const stored = localStorage.getItem(WORKFLOWS_STORAGE_KEY);
    if (stored) {
      const workflows = JSON.parse(stored) as Workflow[];
      setSavedWorkflows(workflows);
      return workflows;
    }
  } catch (e) {
    console.error("Failed to load workflows:", e);
  }
  return [];
};

const persistWorkflows = (workflows: Workflow[]) => {
  try {
    localStorage.setItem(WORKFLOWS_STORAGE_KEY, JSON.stringify(workflows));
  } catch (e) {
    console.error("Failed to save workflows:", e);
  }
};
```

## Variable Priority

When the same variable name exists in multiple sources:

1. **Workflow variables** (from previous steps) - Highest priority
2. **Environment variables** - Lower priority
3. **Undefined** - Variable placeholder remains if not found

```typescript
// If both exist:
// Environment: baseUrl = "https://api.prod.example.com"
// Workflow: Login.baseUrl = "https://api.dev.example.com"

// Using {{Login.baseUrl}} → "https://api.dev.example.com" (workflow wins)
// Using {{baseUrl}} → "https://api.prod.example.com" (environment)
```

## Troubleshooting

<Accordion title="Variable not being replaced">
  **Check:**
  1. Variable was extracted in a previous step (check execution log)
  2. Step name matches exactly (case-sensitive): `{{Login.token}}` not `{{login.token}}`
  3. Variable name matches: `{{Login.userId}}` not `{{Login.user_id}}`
  4. JSONPath is correct (test the step to preview extraction)
</Accordion>

<Accordion title="JSONPath extraction returns undefined">
  **Common issues:**
  1. Path doesn't start with `$`: Use `$.data.id` not `data.id`
  2. Case mismatch: `$.userId` vs `$.UserId`
  3. Array index out of bounds: `$.items[5]` when only 3 items
  4. Response body is not JSON (check step response)
  
  **Test extraction:**
  - Click "Test Request" on the step
  - View response body
  - Verify JSONPath matches the structure
</Accordion>

<Accordion title="Workflow executes out of order">
  Workflows always execute sequentially (top to bottom). If steps seem out of order:
  1. Check the step `order` values (should be 0, 1, 2, ...)
  2. Use the ↑ / ↓ buttons to reorder steps
  3. Save and re-run the workflow
</Accordion>

<Accordion title="Request override not working">
  Ensure:
  1. Override section is expanded (click "Edit Body/Headers")
  2. Correct tab is selected (Body / Headers / URL)
  3. JSON is valid (for body overrides)
  4. Variables are properly interpolated: `{{StepName.var}}`
</Accordion>
